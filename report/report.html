Front-end design

Every page was designed to be minimal and clea, this was done not only to keep the website light-weight but also ensure excellent usability.


Use registration and login

User when registering for a new account, is automatically logged in. Already logged in users can setting new accounts as well all within the same page.


Creating post with text and image

For my site, I handle image aspect of posts using third party uploadeders. As only post image displaying was a functional requirement - I decided to focus on it.

Database and tables

Database solution used: Sqlite3.


Database storage and retrieval

All of database requests also including website's cosmetic changes are all done using jQuery's built-in Ajax handler for requests and sqlite3 NodeJS plugin for actual interaction with DB.


HMTL & CSS validation

Since most of my CSS is handled by Bootstrap, I didn't have much to validate but in the case of HTML, most of my warnings were from using ejs related commands in the file.


Selenium Testing

2 tests were ran, one for login and one for testing the comments. Both came out successful as can be seen from the screenshots.


qUnit Testing

2 tests were ran, both login related, to check login status when logged in and when logged out. Both tests were successful.


Security, privacy and legal issues

  • Security: Since my DB was managed by Sqlite3, it auto handles SQL injection so I didn't have to write my own code to prevent any injection related attacks as it was all already built-in. A possible vulnerability is the website in it's current form uses http and not https so all data flowing through the site is not encrypted.
  • Privacy: Since every request is send dynamically with AJAX and nothing is shown in the query strings, users can stay calm as their data is not exposed on the URLs, however the DB in it's current form is not password protected so that's something to look out for.
  • Legal: All code is open sourced for anyone to view on my personal GitHub account.
